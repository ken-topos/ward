# Ward work packages

One file per work package: `<ID>-<slug>.md`, mapped 1:1 to a git branch
`wp/<ID>-<slug>` and a mootup thread. ID letters encode the workstream (`S` seam,
`Tr` translate, `E` engines, `A` architecture). Catalog:
`../03-program-of-work.md`.

Each WP opens with a Steward frame:

> **Owner:** Team · **Size:** S/M/L · **Risk:** ★–★★★ · **Branch:**
> `wp/<ID>-<slug>` · **Feeds:** … → **Gate**.

then Scope, deliverables, acceptance criteria, and guardrails.
